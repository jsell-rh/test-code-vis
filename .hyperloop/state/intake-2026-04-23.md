## NFR constraints on existing tasks

The following NFRs from `specs/prototype/nfr.spec.md` apply as acceptance criteria to existing tasks, not as new tasks.

- **Godot 4 engine** — Tasks 007–014 must target Godot 4.x; all scripts must be GDScript.
- **Python extractor** — Tasks 002–006 must use only the Python standard library and `tree-sitter` (or `ast`); no other third-party dependencies.
- **JSON interface contract** — The JSON produced by task-006 is the only artifact consumed by task-007; neither component has direct knowledge of the other.
- **Desktop platform** — Tasks 007–014 must run natively on Linux (Fedora) without browser, container, or VM.
- **Performance at kartograph scale** — Rendering and camera tasks (008–014) must sustain ≥ 30 fps across kartograph's full scene (~6 bounded contexts, ~50 modules, ~100 files).
- **Prototype disposability** — Code quality is secondary to iteration speed; the JSON scene graph format (task-001) is the only artifact worth preserving if the prototype is discarded.
